## 1. 概述

Point24 是一个高效的 **24 点游戏** 求解工具。输入 4 个数字，程序通过穷举所有排列、运算符组合和二叉树结构，使用 `+ - * /` 四则运算找出所有结果为 24 的表达式，并通过标准化算法消除重复解。

### 核心特性

- **精确计算** — 使用 `Fraction` 分数运算，避免浮点精度丢失
- **完全覆盖** — 5 种二叉树结构，覆盖所有加括号方式
- **智能去重** — 利用交换律/结合律标准化，消除等价表达式
- **多种输入** — 支持命令行参数和交互式输入两种模式
- **无依赖** — 仅使用 Python 标准库

## 5. 算法详解

### 5.1 总体流程

```
输入 4 个数字
    │
    ▼
枚举所有排列 (4! = 24 种)
    │
    ▼
枚举所有运算符组合 (4³ = 64 种)
    │
    ▼
枚举 5 种运算树结构
    ├── (((a○b)○c)○d)   ① 左结合
    ├── ((a○(b○c))○d)   ②
    ├── (a○((b○c)○d))   ③
    ├── ((a○b)○(c○d))   ④ 两两分组
    └── (a○(b○(c○d)))   ⑤ 右结合
    │
    ▼
计算结果是否等于 24？
    │
    ├── 是 → 标准化去重 → 输出表达式
    └── 否 → 跳过
```

### 5.2 分数运算 — 避免精度丢失

```python
def calc(x, y, op):
    if op == '/':
        return Fraction(x, y)  # 精确分数运算
```

浮点数计算 `8 / (3 - 8/3)` 会得到 `23.999...` 而非精确的 24，使用 `Fraction` 确保判断准确。

### 5.3 5 种二叉树结构

4 个数字的运算顺序对应 5 种不同的加括号方式（Catalan 数 C₃ = 5）：

| 编号 | 二叉树形态 | 表达式示例 |
|------|-----------|-----------|
| ① | `(((ab)c)d)` | `((1+2)+3)*4` |
| ② | `((a(bc))d)` | `(1+(2+3))*4` |
| ③ | `(a((bc)d))` | `1*((2+3)*4)` |
| ④ | `((ab)(cd))` | `(1+2)*(3+4)` |
| ⑤ | `(a(b(cd)))` | `1+(2+(3*4))` |

程序通过 `TEMPLATES`（求值）和 `TREES`（建树）两套 lambda 一一对应处理。

### 5.4 表达式去重 — canonical() 标准化

利用**交换律**和**结合律**将表达式转为规范形式，消除重复解。

**加减法标准化：** `_get_terms()` 将表达式展开为 `[(符号, 项), ...]` 列表，然后排序重组。

```
(1 + 2 + 3) * 4  →  项列表: [(+, 1), (+, 2), (+, 3), (+, 4)]
(3 + 2 + 1) * 4  →  同上 → 视为重复解
```

**乘除法标准化：** `_get_factors()` 将表达式展开为 `[(分子/分母, 因子), ...]` 列表，排序重组。

```
1 * 2 * 3 * 4  →  因子列表: [(分子, 1), (分子, 2), (分子, 3), (分子, 4)]
4 * 3 * 2 * 1  →  同上 → 视为重复解
```

### 5.5 智能括号 — to_string() 优先级处理

根据运算符优先级决定是否添加括号：

| 表达式 | 输出 | 说明 |
|--------|------|------|
| `(+, a, (*, b, c))` | `a + b * c` | 乘法优先级高，无需括号 |
| `(*, a, (+, b, c))` | `a * (b + c)` | 加法优先级低，需要括号 |
| `(-, a, (-, b, c))` | `a - (b - c)` | 右结合性，括号不可省略 |
| `(/, a, (/, b, c))` | `a / (b / c)` | 右结合性，括号不可省略 |

## 6. 函数 API

### `calc(x, y, op)`

执行二元运算。

| 参数 | 类型 | 说明 |
|------|------|------|
| `x` | int / Fraction / None | 左操作数 |
| `y` | int / Fraction / None | 右操作数 |
| `op` | str | 运算符：`'+'`, `'-'`, `'*'`, `'/'` |
| **返回** | int / Fraction / None | 运算结果，除零时返回 `None` |

### `to_string(node, parent_op=None, side=None)`

将表达式树转换为数学字符串。

| 参数 | 类型 | 说明 |
|------|------|------|
| `node` | tuple / int / str | 表达式树节点 |
| `parent_op` | str / None | 父节点运算符 |
| `side` | str / None | 当前节点在父节点中的位置：`'left'` / `'right'` |
| **返回** | str | 可读的数学表达式 |

### `canonical(node)`

将表达式转为规范形式（用于去重）。

| 参数 | 类型 | 说明 |
|------|------|------|
| `node` | tuple | 表达式树 |
| **返回** | str | 标准化后的字符串 |

### `point24_solve(numbers)`

求解 24 点，返回所有去重后的解。

| 参数 | 类型 | 说明 |
|------|------|------|
| `numbers` | list[int] | 4 个数字 |
| **返回** | list[str] | 表达式字符串列表 |

### `parse_args()`

解析命令行参数。

| 返回 | 说明 |
|------|------|
| `argparse.Namespace` | 包含 `numbers` 属性 |

### `main()`

程序入口，读取输入、求解并输出结果。

## 9. 常见问题

### Q: 输入的数字可以重复吗？

可以。例如 `8 8 3 3` 中的 8 和 3 各出现两次。

### Q: 输入的数字超过 4 个会怎样？

程序只取前 4 个数字进行求解。

### Q: 无解时输出什么？

程序输出 `Total 0 results`。

### Q: 为什么 `1 1 1 1` 没有解？

因为 `1 + 1 + 1 + 1 = 4`，无法通过四则运算得到 24。

### Q: 结果中的表达式顺序是固定的吗？

否，结果顺序不固定，取决于枚举顺序。
